# Kotlin JNI scaffold for wsjt-ffi

A minimal end-to-end example: Kotlin class → JNI C shim → libwsjt.so.
Works on Android (NDK) and desktop JVM unchanged.

## Files

| File                        | Purpose                                                   |
|-----------------------------|-----------------------------------------------------------|
| `kotlin/Wsjt.kt`            | Kotlin wrapper with `open / decode / close` API           |
| `native/wsjt_jni.c`         | JNI shim marshalling ShortArray / Message<-> C ABI        |

The wrapper is ~85 lines of Kotlin and the shim is ~115 lines of C.
No JNA, no autogenerated bindings — explicit and easy to audit.

## Build (desktop JVM, Linux x86_64)

Useful for unit tests and headless integration. Requires a JDK with
`JAVA_HOME` set.

```bash
# 1. Build libwsjt.so (Rust side)
cargo build -p wsjt-ffi --release

# 2. Compile the JNI shim against the Rust library
cd wsjt-ffi/examples/kotlin_jni
gcc -shared -fPIC -o libwsjt_jni.so native/wsjt_jni.c \
    -I"$JAVA_HOME/include" -I"$JAVA_HOME/include/linux" \
    -I../../include \
    -L../../../target/release -lwsjt \
    -Wl,-rpath,'$ORIGIN:'"$PWD/../../../target/release"

# 3. Compile + run the Kotlin snippet (kotlinc must be on PATH)
kotlinc kotlin/Wsjt.kt -include-runtime -d wsjt.jar
java -Djava.library.path="$PWD:$PWD/../../../target/release" -cp wsjt.jar \
     -e 'println(io.github.rsft8n.Wsjt.version)'
```

## Build (Android, arm64)

For an Android app, add `libwsjt.so` and `libwsjt_jni.so` under
`src/main/jniLibs/arm64-v8a/` and consume `Wsjt.kt` from Kotlin code.

```bash
# Set the NDK version-qualified compiler
export NDK_CLANG=$ANDROID_NDK_HOME/toolchains/llvm/prebuilt/linux-x86_64/bin/aarch64-linux-android30-clang

# 1. Build libwsjt.so for arm64 (Rust side)
rustup target add aarch64-linux-android
CC_aarch64_linux_android=$NDK_CLANG \
CARGO_TARGET_AARCH64_LINUX_ANDROID_LINKER=$NDK_CLANG \
    cargo build -p wsjt-ffi --release --target aarch64-linux-android

# 2. Compile the JNI shim for the same ABI
$NDK_CLANG -shared -fPIC -o libwsjt_jni.so \
    native/wsjt_jni.c \
    -I../../include \
    -L../../../target/aarch64-linux-android/release -lwsjt
```

Then in your Gradle module:

```kotlin
// app/build.gradle.kts
android {
    ndkVersion = "27.0.11902837"
    defaultConfig {
        ndk { abiFilters += "arm64-v8a" }
    }
    sourceSets["main"].jniLibs.srcDirs("src/main/jniLibs")
}
```

## Memory model

- `Wsjt.open(...)` allocates a native handle; `close()` (or `use { }`)
  releases it.
- `decode()` internally allocates, populates, and frees a
  `WsjtMessageList` on every call — Kotlin side sees only `List<Message>`.
- No Kotlin-visible pointer / handle besides the opaque `Long` stashed
  inside the `Wsjt` instance; attempting `decode()` after `close()`
  throws `IllegalStateException`.

## Design notes

- The shim returns `Array<String>` with a `|`-separated payload rather
  than a JNI struct-per-message. Simpler to keep in sync with Rust
  changes (only one JNI signature to maintain). For very hot paths
  consider a per-field `DirectByteBuffer` layout.
- Default protocol is `Protocol.FT8`; switch to `Protocol.FT4` for
  7.5 s slots.
- Thread safety: one `Wsjt` instance is **not** `Sync`. Each worker
  thread should `open()` its own handle. `lastError()` uses
  thread-local storage on the Rust side so concurrent decoders do not
  interfere.
