# rs-ft8n PWA 運用マニュアル

**[English version](manual.en.md)** | **[アプリを開く](https://jl1nie.github.io/rs-ft8n/)**

rs-ft8n はブラウザだけで動作する FT8 デコーダ PWA です。インストール不要で、Chrome / Edge / Safari (WebKit) で動作します。

## 初回セットアップ

1. [アプリ](https://jl1nie.github.io/rs-ft8n/) を開く
2. 設定パネル（右上の歯車アイコン ⚙）が自動で開く
3. 以下を入力:
   - **My Callsign** — 自局コールサイン（例: `JA1ABC`）
   - **My Grid** — グリッドロケータ（例: `PM95`）
   - **Audio Input** — USB オーディオインタフェース（受信側）を選択
   - **Audio Output** — TX 音声の出力先デバイスを選択（リグへの送信側）
4. **Start Audio** をタップ → ライブデコード開始

> CAT 制御を使う場合は、CAT Protocol を選び **Connect CAT** をタップ。Web Serial API 対応ブラウザが必要です（Chrome / Edge）。

## 2 つのモード

### Scout モード — ゆるい CQ 運用

メッセージアプリのようなチャット UI で、気軽に QSO できます。移動運用やスマホでの使用に最適。

**基本操作:**

| 操作 | 動作 |
|------|------|
| **CQ ボタン** | CQ を発信 |
| **ウォーターフォールをタップ** | TX 周波数（DF）を設定。赤い縦線で表示 |
| **受信メッセージをタップ** | その局を呼ぶ（自動で QSO 開始） |
| **Auto チェック** | ON: 自動応答。OFF: TX メッセージを手動選択 |
| **Halt** | 送信を即座に停止 |

**チャット表示の見方:**

- 青枠の行 = 受信メッセージ（RX）
- グレー枠の行 = 送信メッセージ（TX）
- **黄色太字** = 自局コールサイン
- **緑太字** = QSO 相手のコールサイン
- 左端の数字 = SNR (dB)

**QSO の流れ（自動モード）:**

1. CQ ボタンで CQ 発信、または受信メッセージをタップして呼ぶ
2. 相手から応答 → 自動でレポート交換
3. RR73 / 73 → QSO 完了（ログに自動保存）

---

### Snipe モード — DX ハンティング

ターゲット局を追い込む専用モード。ウォーターフォールが大きく表示され、500 Hz BPF 窓の位置が可視化されます。

#### Watch フェーズ（全帯域受信）

ターゲットを探し、呼ぶ周波数を決めるフェーズ。

| 操作 | 動作 |
|------|------|
| **DX Call (AP)** を設定パネルで入力 | ターゲット局を指定（AP デコードが有効化） |
| **ウォーターフォールをタップ** | 500 Hz 窓の中心周波数を設定 |
| **受信メッセージをタップ** | その局をターゲットに設定 |

**Watch 画面の表示:**

- **上部**: ターゲット局の最新メッセージ（コール / 周波数 / SNR）
- **Callers**: ターゲットを呼んでいる他局の一覧
- **下部**: 全バンドの受信メッセージ一覧
- **QSO 進捗ドット**: ●○○○ → ●●○○ → ●●●○ → ●●●●

#### Call フェーズ（呼び続ける）

Watch で DF を決めたら **Call** ボタンで切り替え。

- 自分とターゲットの交信だけが表示される（ノイズ削減）
- 自動で `callStation()` が発動し、呼び続ける
- QSO 失敗（リトライ上限）→ 自動で Watch に戻る
- 手動で Watch に戻って DF を変更することも可能

**Watch / Call の切り替え:**

Snipe ビュー上部の `[Watch] [Call]` タブで切り替え。

## ウォーターフォール

200-2800 Hz のリアルタイムスペクトログラム。

| 要素 | 説明 |
|------|------|
| **上部ラベル** | 周波数軸（200, 500, 1000, 1500, 2000, 2500 Hz） |
| **赤い点線（縦）** | Scout モードの DF（TX 周波数） |
| **水色の帯** | Snipe モードの 500 Hz 窓 |
| **赤い点線（横）** | ピリオド境界（15 秒区切り） |
| **黄色テキスト** | デコードされたメッセージ |
| **タップ / クリック** | DF / 窓の中心周波数を設定 |

## QSO ステートマシン

4 段階の状態遷移で QSO を管理:

```
IDLE → CALLING → REPORT → FINAL → IDLE (完了)
```

| 状態 | 意味 | 送信内容 |
|------|------|----------|
| **IDLE** | 待機中 | — |
| **CALLING** | 呼び出し中 | `DX MYCALL GRID` or `CQ MYCALL GRID` |
| **REPORT** | レポート交換中 | `DX MYCALL R+00` |
| **FINAL** | 確認待ち | `DX MYCALL RR73` or `73` |

- **Auto ON**: 状態遷移は全自動。応答がなければリトライ（最大 15 回）。
- **Auto OFF**: TX メッセージ選択ボタンが表示され、手動で選択。
- リトライ上限に達すると QSO は incomplete としてログに記録。

## 設定パネル

歯車アイコン ⚙ で開閉。

| 項目 | 説明 |
|------|------|
| **My Callsign** | 自局コールサイン |
| **My Grid** | グリッドロケータ（4 桁） |
| **Audio Input** | マイク / USB オーディオを選択 |
| **Audio Output** | TX 音声の出力先デバイスを選択（デフォ���ト: システム出力） |
| **DX Call (AP)** | ターゲット局コール（AP デコード + Snipe ターゲット） |
| **CAT Protocol** | Yaesu (FTDX10) / Icom (CI-V) |
| **Connect CAT** | Web Serial でリグに接続（PTT 制御） |
| **Start / Stop Audio** | ライブデコードの開始・停止 |
| **Reset QSO** | QSO を中断（incomplete としてログに保存） |
| **Multi-pass subtract** | 逐次干渉除去（3 パス SIC）の有効/無効 |

## ログ管理

### 自動保存

- **QSO 完了時**: 自動で localStorage に保存
- **QSO 中断時**: Reset ボタン or リトライ上限で incomplete として保存（状態タグ付き）
- **全受信メッセージ**: 全デコード結果を RX ログとして蓄積（最大 10,000 件）

### ZIP エクスポート

設定パネルの **Export ZIP (ADIF + RX)** ボタンで、以下の 3 ファイルを ZIP ダウンロード:

| ファイル | 内容 |
|----------|------|
| `qso_complete_YYYYMMDD.adi` | 完了 QSO のみ（LoTW / HAMLOG 投入用） |
| `qso_all_YYYYMMDD.adi` | 全 QSO（incomplete には `<COMMENT>incomplete:STATE</COMMENT>` 付き） |
| `rx_YYYYMMDD.csv` | 全受信デコードログ（UTC, Freq, SNR, Message） |

### ログのクリア

**Clear All Logs** ボタンで QSO ログと RX ログの両方を削除（確認ダイアログあり）。

## WAV ファイルでのオフライン解析

ライブオーディオを使わずに、録音済み WAV ファイルを解析できます。

1. ライブデコード停止中にドロップゾーンに WAV をドラッグ＆ドロップ（または「select file」をクリック）
2. **要件**: 12 kHz / 16 bit / mono の WAV
3. ウォーターフォール + デコード結果が即座に表示

---

## CAT 制御

Web Serial API を使い、ブラウザからリグの PTT を制御します。

**対応プロトコル:**

| プロトコル | リグ例 | ボーレート |
|-----------|--------|-----------|
| **Yaesu** | FTDX10, FT-991A 等 | 38400 |
| **Icom CI-V** | IC-7300, IC-705 等 | 19200 |

**接続手順:**

1. USB ケーブルでリグと PC を接続
2. 設定パネルで CAT Protocol を選択
3. **Connect CAT** → ブラウザのシリアルポート選択ダイアログが開く
4. ポートを選択 → 接続完了
5. TX 時に自動で PTT ON/OFF

> Web Serial API は Chrome / Edge でのみ利用可能です。Safari / Firefox は非対応。

## トラブルシューティング

| 症状 | 対処 |
|------|------|
| 「Select audio device」と表示される | 設定パネルで Audio Input を選択してください |
| デコード結果が 0 件 | アンテナ接続、オーディオレベル、周波数（14.074 MHz 等）を確認 |
| WAV ドロップでエラー | 12 kHz / 16 bit / mono か確認。48 kHz WAV は非対応 |
| CAT 接続できない | Chrome / Edge を使用。ブラウザ再起動後に再試行 |
| ウォーターフォールが真っ黒 | Audio Input が正しいデバイスか確認。Start Audio を押し直す |
| QSO が進まない | Auto チェックが ON か確認。DX Call が正しいか確認 |
